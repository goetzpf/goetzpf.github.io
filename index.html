Welcome to Goetz Pfeiffer’s Project Site

This is a list of open source projects mostly related to my work at the Helmholtz-Zentrum Berlin.

For some projects, I am the only developer, for others I have made contributions.

A third category of projects have no contributions by me but are republished here since they are very old and hard to find anywhere else.

All projects are have some kind of open source license, the copyright holder is described in the project.

Databases

Projects that act as a database.

  • YAML-StructuredData [1]: Software to manage, organize, storing and type-check your data in the YAML format.

Software development

These are tools that in general help software development:

  • git-gq: A patch queue implementation for git. This is inspired by mercurial (mq) patch queues.

  • hg-diff: hg-diff is a program to browse and compare mercurial revisions.

  • hgstat: A shell script to display the overall state of a mercurial repository.

  • hgen [1]: A tool to generate C header files from special comments in C source files, written in perl.

  • pyexpander [1]: A turing complete macro processor with python syntax

  • rsync-dist [1]: This manages the distribution of files, programs and binary data to a server. All old versions are kept and active versions are selected with symbolic links.

Field Bus I/O

  • socan [1]: socan is a high level library for using the CAN Bus with Linux. It also enables MultiCAN to be used on Linux systems.

EPICS Projects

EPICS is a set of software tools and applications which provide a software infrastructure for use in building distributed control systems to operate devices such as Particle Accelerators, Large Experiments and major Telescopes.

The following list shows projects that are used for EPICS:

  • DM2K [2]: A display manager for EPICS. Note that this is legacy code. It is not recommended that you use this display manager for new developments.

  • EDM: A display manager for EPICS. Check out with git clone -b HZB https://github.com/goetzpf/edm.git, then follow instructions in file “README.install” under heading “HZB branch”.

  • Pezca [1]: A Perl interface to channel access, the EPICS network protocol.

  • epics-configure-iptables [1]: A script that configures iptables to convert UDP broadcasts to a given port to UDP unicasts. This solves the problem with UDP unicasts when you run several channel access servers on a single host.

  • sumo [1]: The support module manager sumo is a program to build consistent sets of EPICS support modules and use them in EPICS applications.

  • sumo-free-database [1]: This project contains a dependency database for sumo with some common EPICS device supports published by various institutions.

  • udp-broadcaster [1]: This program receives udp broadcast packets on one interface and port and forwards these to one or more other interfaces. It is used at the Helmholtz-Zentrum Berlin to forward EPICS name resolution requests to other networks for the channel access gateway infrastructure.

EPICS Device Supports

Projects that implement interfaces to certain hardware for EPICS.

  • almLib [1]: An EPICS high resolution timer and alarm clock library for Linux and RTEMS.

  • apps_generictemplate [1]: An EPICS support module to create EPICS template and panel files for applications at the Helmholtz-Zentrum Berlin

  • apps_iocwatch: EPICS IOC statistics support for Linux PCs and for the mvme162, mvme2100 and mvme5500 CPUs running with RTEMS.

  • apps_powersupplies [1]: An EPICS support module to control power supplies at the Helmholtz-Zentrum Berlin

  • bspdep-timer [1]: A timer device support needed by almLib [1]. This is a high resolution timer for Linux PCs and the mvme2100 or mvme5500 VME bus cpu card with the RTEMS operating system.

  • bspdep-vmetas [1]: Implements VME test-and-set functions for the RTEMS operating system for VME CPU boards mvme2100, mvme5500, mvme6100.

  • misc_dbc [1]: An EPICS device support that provides some assertion macros, nothing else.

  • MultiCAN [1]: An EPICS device support for the CAN Bus. Supports the operating systems RTEMS and Linux. For Linux support see also socan.

  • pyphoebusalarm [1]: A Python package to create alarm configurations for Phoebus.

  • EPICS sequencer [2]: The Sequencer implements a language for programming state machines with the EPICS framework.

  • soft_devhwclient [1]: A special record for MultiCAN.

  • vdrive [1]: vdrive provides virtual drives. These are simulated motor drives that can be used in an application that simulates devices that have motors that need to be controlled.

RTEMS support

The operating systems RTEMS is an open source real time operating system.

At the Helmholtz-Zentrum Berlin RTEMS is used on VME Bus systems in the control system.

The following list shows projects that help with RTEMS software development:

  • rtems-from-scratch [1]: This helps to build the RTEMS cross compiler and other development tools. Very old RTEMS versions like version 4.9 are abandoned on the RTEMS project and the cross development tools can no longer be built directly. This project allows to create the cross development tools for RTEMS 4.9, 4.10, 4.11 and 5.1.

  • rtems-build-tool [1]: This helps to build a RTEMS boot image for VME Bus CPUs at the Helmholtz-Zentrum Berlin. For RTEMS 4.9 it also compiles cexp and GeSys in the boot image which implement an interactive shell and support for dynamic loading of modules.

RTEMS sources

These are used by rtems-build-tool which downloads them for you. These projects are here just for completeness.

  • A modified RTEMS kernel: This is the original RTEMS kernel repository with some extra branches (4.9.6-HZB and 4.10.2-HZB) that contain additional patches developed at the Helmholtz-Zentrum Berlin. This repository is automatically fetched when you use rtems-build-tool, so you usually don’t have to download this yourself.

  • RTEMS cexpsh: A shell with support for dynamic module loading for RTEMS 4.9. This repository is automatically fetched when you use rtems-build-tool, so you usually don’t have to download this yourself.

  • RTEMS libbspext: Extended board support for RTEMS 4.9 on mvme2100 VME bus cpus. This repository is automatically fetched when you use rtems-build-tool, so you usually don’t have to download this yourself.

  • RTEMS bootlib [1]: Boot parameter support for RTEMS on VME Bus CPUs. This repository is automatically fetched when you use rtems-build-tool, so you usually don’t have to download this yourself.

  • RTEMS GeSys: A package that together with RTEMS cexpsh creates a bootable image for RTEMS on VME bus cpus for RTEMS version 4.9. This repository is automatically fetched when you use rtems-build-tool, so you usually don’t have to download this yourself.

Undulator control

This is software used to control Undulators at the BESSY synchrotron facility at the Helmholtz-Zentrum Berlin.

  • IDCP [1]: The insertion device control program. This program runs on Linux and RTEMS.

  • IDCP Documentation [1]: This is comprehensive documentation of Undulator control at BESSY and related programs.

  • id-db [1]: The parameter database for IDCP, the insertion device control program.

  • uniserv [1]: The program that runs on the UniDrive motor control device that is used for the BESSY Undulators.

  • unisim [1]: unisim emulates a set of UniDrive motor control devices that run the uniserv program.

Miscellaneous

Projects that don’t fit another category.

  • bash-dirstack: Implements a flexible directory stack for the bash shell.

  • bii_scripts [1]: A collection of scripts for generic software development and for working with EPICS.
